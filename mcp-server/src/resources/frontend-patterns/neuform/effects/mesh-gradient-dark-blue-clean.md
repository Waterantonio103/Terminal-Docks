# Mesh Gradient Dark Blue Clean

Use this Neuform-derived pattern only when it strengthens the accepted product/design direction. It is a heavy recommendation when matched, but never mandatory and never a replacement for accepted specs.

## Intent

Create a dark-blue clean design system built around a shader-like mesh gradient background, premium glass-border shells, and restrained planetary-tech atmosphere.

## Use When

- The accepted DESIGN.md calls for this exact visual move or a close equivalent.
- A visual-first marketing or brand surface needs stronger art direction.
- The project can support canvas/WebGL implementation and reduced-motion fallback.

## Avoid When

- It conflicts with user-provided PRD.md, DESIGN.md, structure.md, screenshots, or brand rules.
- It would replace real product proof with decoration.
- It makes text, controls, or CTAs harder to read or use.
- The project cannot support a reduced-motion fallback or visual QA pass.
- The requested surface is a restrained dense product UI and this effect would add noise.

## Pattern Guidance

- Mesh Gradient Dark Blue Clean Skill
- Apply this as a full design-system direction across background rendering, hero shell, navigation, floating nodes, framed sections, and motion.
- Use it when the interface should feel futuristic, premium, and clean, with a dark-blue mesh-gradient atmosphere powering the whole composition.
- This is not an airy blue page and not generic dark glassmorphism. The mesh gradient should act as the visual engine inside a disciplined, minimal system shell.
- Visual target:
- Build the page on a near-black foundation with a deep navy or steel-blue undertone rather than flat black.
- Use a large primary shell or hero card wrapped in a subtle white-to-transparent border gradient, with a darker inner surface holding the content.
- Place a shader-like mesh gradient, CPPN-style field, or abstract WebGL blue veil behind the content inside that shell so the background feels alive and dimensional.
- Keep typography crisp and bright: white headlines, gray-blue support copy, and very restrained accent states.
- Let the overall tone feel planetary and infrastructural, with hints of edge networks, nodes, or system topography without turning the page into a crowded dashboard.

When selected, translate this pattern into exact DESIGN.md fields: colors, typography, layout, spacing, surfaces, component recipes, motion, responsive behavior, asset requirements, and builder handoff. Do not merely name the effect.
